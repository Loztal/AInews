# Frontend Patterns & Security

## XSS Prevention (Critical)
ALWAYS use `escapeHtml(text)` before inserting into `.innerHTML`:
```javascript
card.innerHTML = '<div>' + escapeHtml(item.title) + '</div>';
```
NEVER: `card.innerHTML = '<div>' + item.title + '</div>';`

## JavaScript Conventions
- All code wrapped in IIFE: `(function () { 'use strict'; ... })();`
- Use `var` (ES5 compat), not `let`/`const`
- DOM queries once, reuse references — never query in loops
- `localStorage` access always in try/catch
- Use `fetch()` API with `.then()` chains, not async/await

## CSS Conventions
- All colors use CSS custom properties (`var(--bg)`, `var(--text)`, etc.)
- Dark mode via `@media (prefers-color-scheme: dark)` overriding `:root` vars
- Category-specific colors are hardcoded brand colors in `app.js` (not CSS variables)
- Mobile-first: system font stack, safe area insets, touch targets >= 48px

## Generated Files — Never Edit Manually
- `docs/data/briefing.json` — completely regenerated by `python -m scraper.main`

## Service Worker
- Cache version in `sw.js` (`CACHE_VERSION`) must be bumped when app shell files change
- Network-first strategy for `briefing.json`, cache-first for static assets
