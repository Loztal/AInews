# AInews — Claude Code Rules

## Project
Daily Claude ecosystem news aggregator. Python scraper (8 source modules) → `docs/data/briefing.json` → static PWA dashboard.

## Critical Rules
- **Never edit `docs/data/briefing.json`** — it is generated by `python -m scraper.main`. Edit source modules in `scraper/sources/` instead.
- **Never raise exceptions in source fetchers** — catch errors, print a warning, return `[]` or fallback data.
- **Always run tests before committing**: `python -m pytest tests/ -v --tb=short`
- **Every item dict must have**: `title`, `date` (ISO 8601 with TZ or `""`), `url`, `summary`, `source`.

## Adding a New Category
All 3 locations must be updated together:
1. `scraper/main.py` — add to `sources` list (line ~26) and ensure-all-categories list (line ~62)
2. `scraper/summarizer.py` — add to both `category_labels` dicts (lines ~49 and ~74)
3. `docs/js/app.js` — add to `CATEGORY_META` (line ~9) and `CATEGORY_ORDER` (line ~19)

## Code Style
- Python: PEP 8, docstrings on modules and public functions, `logging` with `%s` formatting in main.py, `print()` for warnings in sources
- JavaScript: Vanilla ES5, IIFE wrapped, `escapeHtml()` for ALL user data in `.innerHTML`
- CSS: All colors via `var(--name)`, support both light and dark mode via `prefers-color-scheme`

## Testing
- Framework: pytest + responses (HTTP mocking)
- Fixtures: `tests/fixtures/` (sample XML/HTML feeds)
- Mock HTTP with `@responses.activate`, use `requests.ConnectionError` not builtin `ConnectionError`
- Test classes grouped by behavior: `TestFetch`, `TestDeduplication`, etc.

## Commands
- Run tests: `python -m pytest tests/ -v --tb=short`
- Run scraper: `python -m scraper.main`
- Run single test file: `python -m pytest tests/test_claude_code.py -v`
